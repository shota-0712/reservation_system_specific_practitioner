## 1. フロントエンド構成

| アプリ | 用途 | ユーザー | 技術 |
|--------|------|---------|------|
| Customer App | 予約・マイページ | 顧客（LINEミニアプリ） | **現行: Vanilla HTML/JS**（将来React化可） |
| Admin Web | 管理画面 | 店舗/本社管理者 | Next.js + shadcn/ui |
| Platform Admin | SaaS運営 | 運営者 | Next.js（Adminと共通基盤） |

---

## 2. Customer App（LINEミニアプリ）

### 2.1 主要機能
- メニュー選択
- 施術者選択
- 日時選択
- 予約確認/完了
- マイページ（予約履歴/キャンセル）

### 2.2 テナント/店舗識別
- URLパス `/{store_code}` から店舗を特定
- APIは `/api/v1/{tenantKey}/...` を使用
  - `tenantKey = store_code / tenant_id / tenant_slug`

---

## 3. Admin Web

### 3.1 主要機能
- ダッシュボード（KPI/今日の予約）
- 予約管理（一覧・ステータス更新）
- 顧客管理（CRM）
- メニュー/スタッフ管理
- 店舗設定

### 3.2 マルチ店舗対応
- 店舗セレクターで `store_id` を切替
- HQは全店舗集計を参照可能（`store_id` 未指定で集計）

---

## 4. UI/UX方針

- 予約導線は最短ステップ
- 管理画面は **情報密度と速度** を優先
- モバイル（タブレット）最適化
