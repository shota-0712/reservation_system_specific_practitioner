# カルテ機能設計書

**バージョン**: 2.1.0
**最終更新日**: 2026-01-31
**ステータス**: Active

---

## 1. 目的

- 施術履歴を蓄積し、顧客満足度と再来店率を向上
- スタッフ間の引継ぎを容易にする

---

## 2. データモデル

### 2.1 kartes テーブル

- `customer_id`, `practitioner_id`, `store_id`, `reservation_id` に紐付け
- 写真は Cloud Storage に保存しURLを保持

### 2.2 karte_templates テーブル

- 店舗ごとにカルテ入力項目をカスタマイズ
- JSONBで動的項目を保持

---

## 3. 主要機能

- カルテ作成/更新/閲覧
- 予約詳細からカルテへ遷移
- 顧客詳細から過去カルテ一覧

---

## 4. アクセス制御

- スタッフ: 自店舗カルテの閲覧/作成
- 管理者: 全カルテの閲覧
- RLSで tenant_id を強制
