# Docs Navigation

どこを見るべきか迷ったときのための最小ナビゲーションです。

## 仕様・設計の正本

- `architecture/`  
  システム仕様、設計方針、セキュリティ設計、データモデル定義。

## 運用手順・実行ログ

- `runbooks/`  
  移行計画、ガードレール、カットオーバー手順、実行ログ。

## 実装時の参照文献（バックエンド）

- `backend-development/`  
  API設計・認証・スキーマ設計・移行運用など、実装時の判断に使う参照資料。  
  入口は `backend-development/README.md`。

## まず読む順序

1. 仕様確認: `architecture/`
2. 実装判断: `backend-development/`
3. 運用反映: `runbooks/`
